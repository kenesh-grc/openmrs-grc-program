# Incident Response Plan – OpenMRS

## 1. Purpose
This Incident Response Plan defines the procedures for identifying, responding to, and recovering from security incidents involving OpenMRS systems handling electronic Protected Health Information (ePHI).

## 2. Scope
This plan applies to:
- OpenMRS application
- AWS cloud infrastructure
- Databases storing ePHI
- Employees, contractors, and third-party vendors

## 3. Incident Response Team
| Role | Responsibility |
|-----|---------------|
| Incident Response Lead | Overall coordination |
| GRC Analyst | Compliance, documentation |
| IT Operations | Technical containment |
| Legal/Compliance | HIPAA assessment |
| Communications | Internal/external messaging |

## 4. Incident Classification
| Severity | Description |
|---------|-------------|
| Low | No ePHI exposure |
| Medium | Potential ePHI exposure |
| High | Confirmed ePHI breach |

## 5. Incident Response Phases
### 5.1 Identification
- Alerts from monitoring systems
- User reports
- Third-party notifications

### 5.2 Containment
- Isolate affected systems
- Disable compromised accounts

### 5.3 Eradication
- Remove malware
- Patch vulnerabilities

### 5.4 Recovery
- Restore systems
- Monitor for recurrence

### 5.5 Lessons Learned
- Root cause analysis
- Control improvements

## 6. HIPAA Breach Notification
Incidents involving ePHI are evaluated under HIPAA Breach Notification Rule (45 CFR §164.400–414).

## 7. Documentation & Evidence
- Incident ticket
- Timeline
- Evidence collected
- Final incident report

## 8. Plan Review & Maintenance
Reviewed annually or after major incidents.
